# AI Usage

I acknowledge that AI tools are now commonly used in the tech field and can be helpful in development. However, I intentionally avoided relying on AI for core thinking tasks such as debugging and logical decision-making, as I wanted to fully understand and build the program on my own.

Most of the logic, structure, and problem-solving in this project were done independently. I only used AI in specific cases where I needed a second perspective on parts that were not immediately visible during testing, such as missing requirements or small details that are easy to overlook even after running the program.

In those situations, I used AI more as a guide for checking completeness and improving clarity rather than for generating the main logic of the program. After that, I personally reviewed and ensured that everything was correctly implemented and that I understood how each part works.

This project is also a quiz activity designed for my learning and skill development. I treated it as an opportunity to gain real experience in programming, and I believe relying heavily on AI would take away that learning process. Because of that, I made sure to solve most of the problems on my own instead of depending on AI.

To sum it up, AI was used only as a supportive tool for awareness and refinement, while the actual thinking, design, and implementation were done primarily by me.
